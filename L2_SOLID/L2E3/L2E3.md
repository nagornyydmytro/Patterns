# L2E3 (Vehicles)

`Vehicle` з методом `drive()` типу універсальний, але не всі тз можуть нормально його реалізувати

У `Bicycle` метод `drive()` перевизначений так, що кидає помилку, тобто якщо десь у коді очікують `Vehicle` і викликають `drive()`, то для `Bicycle` все падає. Через це `Bicycle` не можна нормально підставляти замість `Vehicle`

Базовий клас намагається дати один набір методів для всіх, хоча для різних типів транспорту потрібні різні, для велосипеда наприклад `ride()`. Тут краще кілька маленьких інтерфейсів, ніж один великий універсальний

Можна не робити в базовому класі методи, які не підходять всім, а зробити різні інтерфейси `Drivable { drive(); }`, `Rideable { ride(); }` і тд, або зробити `Vehicle` якимось нейтральним, а дії винести в інтерфейси або окремі класи

Наслідування використане як спільний контейнер для всього, але реальні об'єкти мають різні можливості, що потребує винятків в методах,правильніше було б розділити інтерфейси/поведінку
